## 三、实验内容及步骤

### 1. Git安装与配置

1. 从Git官网下载最新版本的Git安装包
2. 运行安装程序，按照默认设置完成安装
3. 配置Git用户信息：
   ```bash
   git config --global user.name "明建超"
   git config --global user.email "mingjianchao@example.com"
   ```

### 2. 本地仓库初始化

1. 创建项目目录：`course-checkin-system`
2. 初始化Git仓库：
   ```bash
   cd course-checkin-system
   git init
   ```
3. 创建`.gitignore`文件，排除不需要版本控制的文件：
   ```
   # Compiled class file
   *.class
   
   # Log file
   *.log
   
   # Package Files
   *.jar
   *.war
   *.nar
   *.ear
   *.zip
   *.tar.gz
   *.rar
   
   # Virtual machine crash logs
   hs_err_pid*
   
   # IDE
   .idea/
   .vscode/
   *.iml
   
   # Maven
   target/
   
   # Node
   node_modules/
   ```

### 3. 远程仓库配置

1. 在Gitee上创建新仓库：`course-checkin-system`
2. 关联远程仓库：
   ```bash
   git remote add origin https://gitee.com/mingjianchao/course-checkin-system.git
   ```

### 4. Git Commit规范学习

学习并实践Angular Commit Message规范：

- **feat**: 新功能
- **fix**: 修复bug
- **docs**: 文档更新
- **style**: 代码格式（不影响代码运行的变更）
- **refactor**: 重构（既不修复bug也不添加功能）
- **perf**: 性能优化
- **test**: 添加测试
- **chore**: 构建过程或辅助工具的变动

### 5. 编码规范调研

调研《阿里巴巴Java开发手册》，重点关注以下方面：

1. **命名规范**
2. **代码格式**
3. **OOP规约**
4. **集合处理**
5. **控制语句**
6. **注释规约**

### 2. 编码规范对比报告

#### 2.1 命名规范

**反面示例（不规范）：**
```java
// 变量名使用拼音
String xuehao = "240104126";
int shuliang = 0;

// 方法名不清晰
public void chuli(User u) {
    // 处理逻辑
}

// 常量名不规范
public static int MAX = 100;
```

**正面示例（规范）：**
```java
// 变量名使用有意义的英文
String studentId = "240104126";
int count = 0;

// 方法名清晰表达意图
public void processUser(User user) {
    // 处理逻辑
}

// 常量名全大写，单词间用下划线
public static int MAX_COUNT = 100;
```

#### 2.2 代码格式

**反面示例（不规范）：**
```java
public class UserController{
public User getUser(Long id){User user=userService.findById(id);return user;}
}
```

**正面示例（规范）：**
```java
public class UserController {

    public User getUser(Long id) {
        User user = userService.findById(id);
        return user;
    }
}
```

#### 2.3 OOP规约

**反面示例（不规范）：**
```java
// 所有属性使用public
public class User {
    public Long id;
    public String username;
    public String password;
}

// 直接访问对象属性
user.password = "123456";
```

**正面示例（规范）：**
```java
// 使用private属性，提供getter/setter
public class User {
    private Long id;
    private String username;
    private String password;
    
    // getter/setter方法
    public String getUsername() {
        return username;
    }
    
    public void setUsername(String username) {
        this.username = username;
    }
}

// 通过方法访问属性
user.setPassword("123456");
```

#### 2.4 集合处理

**反面示例（不规范）：**
```java
// 使用HashMap时未考虑并发
Map<String, User> userMap = new HashMap<>();
userMap.put("key", user);

// 遍历时使用索引
for (int i = 0; i < list.size(); i++) {
    User user = list.get(i);
}
```

**正面示例（规范）：**
```java
// 使用ConcurrentHashMap处理并发
Map<String, User> userMap = new ConcurrentHashMap<>();
userMap.put("key", user);

// 使用增强for循环
for (User user : list) {
    // 处理用户
}
```

#### 2.5 控制语句

**反面示例（不规范）：**
```java
// if语句不加括号
if (user != null)
    return user;

// switch语句没有default
switch (status) {
    case "ACTIVE":
        // 处理
        break;
    case "CLOSED":
        // 处理
        break;
}
```

**正面示例（规范）：**
```java
// if语句必须加括号
if (user != null) {
    return user;
}

// switch语句必须有default
switch (status) {
    case "ACTIVE":
        // 处理
        break;
    case "CLOSED":
        // 处理
        break;
    default:
        // 默认处理
        break;
}
```

## 五、实验总结和思考

### 收获与体会

1. **掌握了Git版本控制**：学会了Git的基本操作，包括初始化仓库、添加文件、提交更改、关联远程仓库等。理解了版本控制在团队协作中的重要性。

2. **理解了编码规范的重要性**：通过调研《阿里巴巴Java开发手册》，认识到规范的代码不仅易于阅读和维护，还能减少bug的产生。

3. **建立了代码质量意识**：学会了识别代码中的技术债，理解了"代码是写给人看的，顺便让机器执行"的道理。

4. **实践了Git Commit规范**：学会了使用规范的提交信息，使代码变更历史更加清晰可追溯。

### 成功与失败原因分析

**成功原因：**
- 认真学习了Git的基本操作和编码规范
- 在实践中不断总结经验
- 使用静态代码检查工具辅助代码质量提升

**遇到的问题：**
- 初期对Git的分支管理理解不够深入
- 编码规范的记忆需要时间积累

**解决方案：**
- 通过实际项目练习加深理解
- 将常用规范整理成checklist，便于查阅

### 改进方向

1. 继续深入学习Git的高级功能，如分支管理、合并冲突解决等
2. 将编码规范内化为编程习惯
3. 学习使用更多的静态代码检查工具
4. 在团队项目中实践代码评审
