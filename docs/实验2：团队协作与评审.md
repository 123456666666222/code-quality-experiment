- 每个模块有明确的负责人
- 定期进行进度同步和问题讨论

### 2. Git分支管理策略

采用Git Flow分支管理策略：

```
main (生产分支)
  ↑
develop (开发分支)
  ↑
feature/* (功能分支)
  ↑
hotfix/* (热修复分支)
```

**分支命名规范：**
- `feature/user-module`：用户模块功能开发
- `feature/course-module`：课程模块功能开发
- `feature/checkin-module`：签到模块功能开发
- `hotfix/fix-login-bug`：修复登录bug

### 3. 代码评审流程

**评审流程：**
1. 开发者完成功能开发，提交代码到功能分支
2. 创建Pull Request，填写规范的描述
3. 指定评审人员（至少2人）
4. 评审人员进行代码评审，提出修改意见
5. 开发者根据评审意见修改代码
6. 评审通过后，合并到develop分支

**评审检查项：**
- 代码规范是否符合团队标准
- 功能实现是否正确
- 是否有潜在的bug
- 是否有性能问题
- 是否有安全隐患
- 单元测试是否完整

### 4. Pull Request规范

**PR标题格式：**

### 5. 持续集成配置

在Gitee上配置持续集成：

```yaml
# .gitee-ci.yml
stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - cd backend
    - mvn clean package -DskipTests
    - cd ../frontend
    - npm install
    - npm run build
  only:
    - develop
    - main

test:
  stage: test
  script:
    - cd backend
    - mvn test
  only:
    - develop
    - main

deploy:
  stage: deploy
  script:
    - echo "部署到测试环境"
  only:
    - develop
```

### 2. 代码评审实例

**评审的代码：**
```java
// CheckinService.java
public CheckinRecord doCheckin(String checkinCode, Long studentId) {
    // 查找签到会话
    Optional<CheckinSession> sessionOpt = sessionRepository.findByCheckinCode(checkinCode);
    if (sessionOpt.isEmpty()) {
        throw new RuntimeException("签到码无效");
    }

    CheckinSession session = sessionOpt.get();

    // 检查签到会话状态
    if (session.getStatus() != CheckinSession.SessionStatus.ACTIVE) {
        throw new RuntimeException("签到已结束");
    }

    // 检查签到码是否过期
    if (session.getStartTime().plusMinutes(checkinCodeExpireMinutes).isBefore(LocalDateTime.now())) {
        throw new RuntimeException("签到码已过期");
    }

    // ... 其余代码
}
```

**评审意见：**
1. **异常处理**：使用RuntimeException不够规范，应该定义业务异常类
2. **参数校验**：缺少对输入参数的校验
3. **日志记录**：缺少关键操作的日志记录
4. **性能优化**：可以考虑使用缓存提高查询效率

**修改后的代码：**
```java

    // 参数校验

        throw new BusinessException("签到码格式错误");
    }
    if (studentId == null) {
        throw new BusinessException("学生ID不能为空");
    }

    // 查找签到会话
    Optional<CheckinSession> sessionOpt = sessionRepository.findByCheckinCode(checkinCode);
    if (sessionOpt.isEmpty()) {
        log.warn("签到码无效: {}", checkinCode);
        throw new BusinessException("签到码无效");
    }

        log.info("签到已结束, sessionId: {}", session.getId());
        throw new BusinessException("签到已结束");

        log.info("签到码已过期, sessionId: {}", session.getId());
```

## 五、实验总结和思考

### 收获与体会

1. **掌握了代码评审流程**：理解了代码评审的目的和价值，学会了如何进行有效的代码评审。通过评审发现了许多潜在的问题，提升了代码质量。

2. **理解了团队协作的重要性**：通过团队协作开发，认识到良好的沟通和分工是项目成功的关键。Git分支管理策略使得多人协作更加顺畅。

3. **学会了Pull Request规范**：掌握了创建规范PR的方法，使代码变更更加清晰可追溯。

4. **了解了持续集成**：学习了CI/CD的基本概念，理解了自动化构建和测试的重要性。

### 成功与失败原因分析

**成功原因：**
- 团队成员分工明确，责任清晰
- 使用Git分支管理策略，避免了代码冲突
- 定期进行进度同步，及时解决问题
- 代码评审严格，保证了代码质量

**遇到的问题：**
- 初期团队成员对Git操作不熟悉，导致一些操作失误
- 代码评审时意见不一致，需要讨论达成共识
- 时间紧张，部分功能测试不够充分

**解决方案：**
- 组织Git操作培训，提高团队成员的Git技能
- 建立代码评审规范，明确评审标准
- 合理安排时间，优先保证核心功能的质量

### 改进方向

1. **加强自动化测试**：增加单元测试和集成测试的覆盖率，提高代码质量
2. **完善CI/CD流程**：配置更完善的持续集成和持续部署流程
3. **建立代码质量指标**：制定代码质量指标，定期进行评估
4. **加强团队沟通**：建立更有效的沟通机制，提高协作效率

### 对未来工作的启示

1. **代码评审是必要的**：代码评审不仅能发现bug，还能促进知识共享和团队成长
2. **规范很重要**：编码规范、提交规范、分支管理规范等都是团队协作的基础
3. **自动化是趋势**：持续集成、自动化测试、自动化部署是现代软件开发的必备技能
4. **沟通是关键**：团队协作中，良好的沟通能避免很多问题
